设计原则

单一原则 SRP

  • 降低类的复杂度，一个类只负责一项职责，一个方法只做一件事

开闭原则 OCP

  • 对扩展开放，对修改关闭
  • 模块相互独立，新功能使用新代码实现，不要修改旧代码

里式替换原则 LSP

  • 子类不要重写父类的方法
  • 子类用新方法完成新功能

依赖倒置原则 DIP

  • 高层依赖低层的抽象而不是实例
  • 常用的情况是面向接口编程
  • IOC 框架就是以此诞生的

接口隔离原则 ISP

  • 为各个类创建专用的接口

合成复用 CRP

  • 迪米特 LoD

  • 类似三层架构，UI 只跟 BLL 联系，BLL 只跟 DAL 联系，禁止 UI 直接跟 DAL 联系

结构型

  • 本质就是封装 N 层

适配器模式

  • 分为类适配器和对象适配器
  • 类适配器：继承父类和接口
  • 对象适配器：实现接口，涉及对象由依赖注入生成
  • 主要以适配某个组件为主，例如封装 Redis 的通用方法

代理模式

  • 看上去跟适配器模式很像
  • 主要以通用功能为主
