数据库基础理论

发表于 2020-12-31 更新于 2021-04-13 分类于 MySQL

点击查看更多

数据库三范式

  • 第一范式：原子性，一个字段不能包含多个信息，所有关系数据库自带第一范式
  • 第二范式：满足第一范式，一张表只描述一种类型，举例就是一张表不要同时出现产品和订单信息
  • 第三范式：满足一二范式下的消除冗余数据
  • 在数据量庞大下为了避免表联结会适当的添加冗余字段，还有在不能加索引的情况下需要添加一个关键字字段充当伪索引

ACID 四大特性

  • 原子性（Atomicity）
  • 一致性（Consistency）
  • 隔离性（Isolation）
  • 持久性（Durabilyty）

原子性

  • 要么成功要么失败
  • 提交事务时会在 Undo log 生成一条日志记录操作前的数据，如果发生错误就会根据这条日志执行相反的操作，例如 insert 错误就执行 delete

一致性

  • 只有先保证其它三个特性才能保证一致性

持久性

  • 事务一旦提交，对数据的改变是永久性的
  • 提交事务时会在 Redo log 生成一条记录，之后写入数据库缓存，最后同步到硬盘
  • Redo log 是文件追加的形式，所以是有序的，有序比无序快

隔离性

  • 写操作：锁
  • 读写操作：MVCC
