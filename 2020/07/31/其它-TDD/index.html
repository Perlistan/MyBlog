点击查看更多

传统流程的弊端

  • 大部分测试是手动的
  • 业务紧急没时间做测试
  • 无法重构代码，怕破坏原有模块
  • 文档从不更新
  • 学习和维护成本高

什么是 TDD

  • 它是一种敏捷开发
  • 传统流程是先做好项目再统一测试，现在是开发时就编写测试
  • 根据测试编写业务代码能减少维护成本
  • 大多数情况下都是测试行为，也就是 public 方法

测试的三个阶段

  • Arrange：创建对象、 数据、输入
  • Act：执行方法并返回结果
  • Assert：检查结果

TDD 流程

  • 拆分需求
  • 编写测试
  • 验证测试
  • 根据测试代码编写产品代码
  • 验证测试
  • 重构
  • 验证测试

测试分类

  • 单元测试：独立模块的测试
  • 集成测试：包含数据库在内一定范围的联合测试
  • 皮下测试：针对接口层的测试
  • UI 测试：在 UI 层的测试

在 .Net 中使用 xUnit 进行测试

  • 新建 xUnit 测试项目
  • 项目命名规范：被测试项目名 + Test
  • 测试类命名规范：被测试类名 + Test
  • 测试类中每个方法都会走一遍构造方法

如何测试？

  • 使用 new 创建实例调用方法
  • 最后使用 Assert 类进行结果检测

测试分组

  • [Trait(“大类名”,”小类名”)] 此特性可以将各个测试方法分组，要在测试资源管理器中选择特征
  • 一个方法或类可以拥有多个 Trait

忽略测试

  • 在 Fact 上加一个参数 Skip = “xxxx” 即可

自定义输出测试信息

  • 注入 ITestOutputHelper 接口
  • 使用该接口的实例方法 WriteLine 输出信息
